Building This Site: Terminal Aesthetic + Claude Code

April 2026

This portfolio has been on my to-do list for years. Literally years. I kept postponing it — first because I was busy automating 1000+ test cases, then because I was setting up CI/CD pipelines, then because I was leading a Playwright CoE. There was always something more urgent.

But here's the thing: if you spend 8 years making software reliable for others, at some point you need a place that says — this is who I am, and this is what I've built.

$ echo "why terminal?"

Everyone's portfolio looks the same. The same hero section, the same gradient background, the same "Hi, I'm a developer" headline with a waving hand emoji. I didn't want that.

As someone who lives in terminals — running Tosca scripts, Playwright tests, CI/CD pipelines, shell commands — the terminal aesthetic felt natural. It's not a gimmick. It's how I actually work.

The rules were simple:

  • Monospace font (JetBrains Mono)
  • Dark background, single accent color
  • $ prompts as section dividers
  • → arrows for lists
  • No images, no icons, no decorations — pure text
  • Content speaks for itself
$ echo "why minimalist?"

I looked at two sites that shaped my thinking:

  • cure53.de — a cybersecurity firm. No cookies, no trackers, no ads. Just text. The "I'm so good I don't need a fancy site" energy. I wanted that confidence.
  • lynnandtonic.com — a designer who redesigns her portfolio every year (she's on version XIX). Multi-page, clean nav, dark mode. I borrowed the structure.

The result is somewhere in between: terminal-styled but well-organized, minimalist but complete. Every page has a purpose, and nothing is there for decoration.

$ echo "the tech"

Zero dependencies. Zero build tools. Zero frameworks.

  • Plain HTML, CSS, and a tiny bit of vanilla JavaScript (just the dark/light toggle)
  • One CSS file with CSS custom properties for theming
  • GitHub Pages — push to main, live in 60 seconds
  • Works on any machine — no Node, no Ruby, no nothing

This matters because I'm on a company laptop with restrictions. No installing Ruby for Jekyll, no npm build chains. Just files and a browser. The constraint became the feature.

$ echo "built with claude code"

Here's where it gets interesting. I built this entire site in a single session using Claude Code — Anthropic's CLI tool for working with code through conversation.

The workflow:

  • Started with brainstorming — described what I wanted, explored design directions visually
  • Narrowed down to the terminal aesthetic through a structured Q&A process
  • Claude Code wrote a full design spec, then an implementation plan
  • Subagents executed each task — repo setup, CSS, pages — with commits after each step
  • I reviewed, gave feedback, and iterated in real-time

What I found: AI doesn't replace your taste or your judgment. It amplifies your ability to execute. I knew what I wanted — the terminal vibe, the content-first approach, the "no fluff" philosophy. Claude Code turned that vision into working code faster than I could have done alone.

This is exactly what I mean when I say "In this AI Era, I'm trying to be a human." — the human brings the vision, the context, the decisions. AI brings the speed.

$ echo "what's next"

This is v.1. It's live, it's real, and it's mine. But it's not done — it never will be. I'll keep updating it the way Lynn Fisher updates hers: as a living document of who I am and what I'm building.

If you're a QA engineer or tester thinking about building your own portfolio — just start. It doesn't need to be fancy. It needs to be you.
